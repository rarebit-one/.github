# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in any rarebit-one repository, please
**do not open a public issue or PR**.

Instead, report it privately by [opening a draft security advisory](https://github.com/rarebit-one/.github/security/advisories/new)
on this repository. **A single advisory here covers reports for any repository
in the org** — you don't need to figure out which repo to file against. (If you
prefer to file directly on the affected repo's Security tab, that also works
and we'll triage from there.)

We treat every report as confidential until a fix is shipped and disclosed
coordinatedly.

### What to include

- The repository (or repositories) affected.
- A description of the issue and its impact.
- Steps to reproduce — ideally a minimal proof-of-concept.
- Any suggested fix or mitigation, if you have one.

### What to expect

- Acknowledgement of your report within 5 business days.
- A working assessment of severity and scope shortly after.
- Coordination on disclosure timing — usually 90 days, sooner for actively
  exploited issues.

### Scope

This policy covers all repositories under the
[rarebit-one](https://github.com/rarebit-one) organization. Vulnerabilities in
upstream dependencies should be reported to those projects directly; we'll
coordinate the host-app fix once an upstream patch is available.
